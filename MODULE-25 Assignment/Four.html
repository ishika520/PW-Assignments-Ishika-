Q4. Explain 5 challenges of file-based storage system which was tackled by DBMS?

Document 5 Challenges of File-Based Storage Systems Tackled by DBMS File-based storage systems, while simple, face significant limitations when handling large and complex datasets. These challenges were addressed by the development of Database Management Systems (DBMS). Here are five key issues:

1. Data Redundancy and Inconsistency

Challenge: File systems often lead to data duplication, as the same information might be stored in multiple files. This redundancy wastes storage space and increases the risk of inconsistencies.
Solution: DBMS ensures data integrity by storing data in a normalized form, eliminating redundancy and reducing the chances of inconsistencies.

Data Isolation

Challenge: Data in file systems is typically isolated within individual files, making it difficult to share and integrate information across different applications or departments.
DBMS Solution: DBMS provides a centralized repository for data, allowing multiple users and applications to access and share information simultaneously.

3. Data Security and Integrity

Challenge: File systems offer limited security and integrity features. Data can be easily lost, corrupted, or accessed by unauthorized users.
Solution: DBMS incorporates robust security mechanisms like user authentication, access controls, encryption, and backup and recovery features to protect data integrity and confidentiality.

4. Data Dependence

Challenge: Applications built on file-based systems are often tightly coupled with the data format, making it difficult to modify or restructure data without affecting the application. DBMS Solution: DBMS promotes data independence by separating data storage and manipulation from application logic. This allows changes to be made to the database structure without impacting applications.

5. Atomicity and Consistency

Challenge: File systems lack support for transactional processing, which is crucial for ensuring data consistency and reliability in complex operations.
DBMS Solution: DBMS provides transaction management capabilities, guaranteeing that data modifications occur as a single, indivisible unit, preserving data integrity and consistency even in case of failures.
